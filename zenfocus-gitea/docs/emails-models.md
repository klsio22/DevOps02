## Boas praticas
- Definir um padrao principal para contas pessoais e manter consistencia.
- Reservar no-reply para envios automaticos.
- Criar aliases de setor que encaminhem para multiplos responsaveis.
- Evitar caixas genericas sem dono responsavel.
- Usar root@zenfocus.com como alias tecnico para time de infraestrutura, sem depender de uma pessoa unica.
- Para separar ambientes, usar por exemplo suporte@dev.zenfocus.com e suporte@homolog.zenfocus.com.
